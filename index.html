What NLPM does

NLPM is a Claude Code plugin that treats natural-language artifacts — skills, agents, commands, rules, hooks, CLAUDE.md — as code. It scores them on a 100-point rubric anchored in 51 rules, checks cross-component consistency, and (optionally) enforces vocabulary discipline against a project's own canonical noun/verb registry.

An auditor pipeline runs against external Claude Code plugins continuously. The dashboard above shows aggregate findings; click any repo row to drill into its per-repo report.

Install

claude plugin install nlpm@xiaolai --scope project

Then in your project:

/nlpm:ls              # discover NL artifacts
/nlpm:score           # 100-point quality scoring
/nlpm:check           # cross-component consistency
/nlpm:vocab-init      # bootstrap a vocabulary skill
/nlpm:report          # render a self-contained HTML report

What's in this site

Auditor dashboard

Cross-repo aggregate: 200+ audited plugins, sortable by score/findings, with rule-distribution bars and a force-directed vocab-drift network.

Per-repo reports

One HTML page per audited repo, linked from the dashboard. Shows the score, findings grouped by rule, drift candidates, vocabulary map (when the repo ships its own registry).

Framework reference

R01–R51, P1–P6, vocabulary concepts, severity levels, drift criteria — all anchored. Every finding in a report links to the rule that triggered it.

Self-contained

Visualizations use AntV G6, vendored. No CDN dependencies; the entire site works offline if you save it.

NLPM is a project by @xiaolai. Reports are regenerated by the auditor pipeline on every audit; the dashboard refreshes daily. Source: github.com/xiaolai/nlpm-for-claude.
